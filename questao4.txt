(V) O algoritmo de Euclides estendido é utilizado para calcular o inverso modular de um número.

COMENTÁRIO: Falso. O inverso modular só existe quando mdc(G, Zn) = 1. O código verifica isso e retorna erro se o MDC for diferente de 1.

(V) A operação (H * inverso) % Zn representa a divisão modular de H por G.
COMENTÁRIO: Verdadeiro. Em aritmética modular, dividir H por G é equivalente a multiplicar H pelo inverso de G.

COMENTÁRIO: Verdadeiro. Quando n1 é primo, o código usa o Pequeno Teorema de Fermat para reduzir o expoente.

COMENTÁRIO: Falso. A função usa exponenciação por quadratura (bit a bit), que é otimizada, não multiplicações diretas.

(V) Quando o resultado do inverso é negativo, o código ajusta o valor somando o módulo m0.
COMENTÁRIO: Verdadeiro. Há a verificação "if (x1 < 0) x1 += m0" que garante que o inverso esteja no intervalo correto.

(V) O cálculo de fi(n1) (função toitente de Euler) é utilizado apenas quando n1 não é primo.
COMENTÁRIO: Verdadeiro. Se n1 é primo, usa Fermat; se é composto mas mdc(a,n1)=1, usa Euler com φ(n1).
